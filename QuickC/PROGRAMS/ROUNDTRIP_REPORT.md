# Отчёт: round-trip тесты QuickC PROGRAMS

Пайплайн (как в `QuickCProgramRoundTripTests`):

```
*.c → QCL (/AS /Gs + SLIBCE.LIB) → Decompiler → MAKE → сравнение EXE
```

Флаги и библиотека совпадают с тестом: `/nologo /AS /Gs`, `SLIBCE.LIB`, модель **small** (`*_S.EXE`).

Эталоны **hello.c**, **add.c** проходят полный цикл.

### Сборка QCL

Все примеры должны собираться через `MAKE all` в `PROGRAMS/` (`/AS /Gs` + `SLIBCE.LIB`).

## 2. Сбой повторной компиляции (QCL / линковка)

### 2.1. Невалидный сгенерированный C

| Файл | Суть примера | Типичные ошибки QCL |
|------|--------------|---------------------|
| **copy.c** | Ручной `while`-копирователь строк (`char *`) | `sub_0010.c`: синтаксис `_psp:[arg]`, `varSS:[65540]`; `main.c`: `printf("%s\n", 624)` вместо строкового литерала |
| **fptr.c** | Запись в видеопамять через `char far *` + `MK_FP` | `varSS:[65532]:[varSS:[65530]] = 65` — far-обращение не сведено к C |
| **glob.c** | Глобальная `counter` + отдельная `bump()` | Неверный синтаксис доступа к глобалам/стеку в `sub_0010` и `main` |
| **jmp.c** | Явные `goto` и метки | В IR попадает текст `div op` (сырое имя операции), QCL: `syntax error before 'div op'` |
| **recur.c** | Рекурсивный `fact()` | То же в `sub_0010.c`: `div op`, неверные условия ветвления |
| **vol.c** | Цикл записи в `MK_FP(0xB800, …)` | Far-индексация `vid[i*2]` → невалидный C с сегментными псевдопеременными |
| **sjmp.c** | `setjmp` / `longjmp` | Сложный CFG + несвязанные `sub_169F`; `div op`, битовые маски с `:` в выражениях |

**Общая причина:** кодогенератор печатает `MemExpr`/сегменты как `seg:[disp]`, не материализует строки/глобалы, не раскрывает `WhileOperation`/деление в идиоматический C.

### 2.2. Линковка: не те runtime-символы

| Файл | Суть примера | Ошибка линкера |
|------|--------------|----------------|
| **mod.c** | `%` и `/` для `int` | Неразрешённые `__aNlrem`, `__aNlshl`, `__aNlshr` — в Makefile только `SLIBCE.LIB`, а декомпилированный код вызывает long-арифметику рантайма |
| **long.c** | `long mix(long, long)` со сдвигами | Неразрешённые `_sub_0025`, `_sub_0037` — внутренние helper'ы 32-битной арифметики не сопоставлены с `.LIB` |

**Интерес:** Makefile восстанавливает флаги модели памяти, но не полный набор библиотек для long/helper-процедур.

## 3. Сбой сопоставления EXE (пересборка OK, байты различаются)

Декомпиляция и `MAKE` завершаются успешно, но эталонный и пересобранный `*_S.EXE` не совпадают побайтово.

| Файл | Суть примера | Заметки |
|------|--------------|---------|
| **switch.c** | `switch` по `int` | Отличие с раннего смещения (0x2) — заголовок/образ; иная структура ветвлений |
| **bits.c** | Битовые поля в `struct` | Упаковка полей, другой порядок операций с масками |
| **dos.c** | `_dos_getdate` | Вызов DOS API + форматная строка |
| **enab.c** | `_disable` / `_enable` | CLI/STI → вызовы `_disable()`/`_enable()`; другой пролог/epilog |
| **int86.c** | `int86(0x21, …)` | `union REGS`, другая раскладка аргументов на стеке |
| **mul.c** | `unsigned` умножение 300×400 | 16-битное MUL, иной код умножения |
| **peek.c** | `peek(seg, off)` из `<dos.h>` | Доступ к абсолютному адресу через runtime |
| **port.c** | `inp` / `outp` (`<conio.h>`) | IN/OUT порты; иная последовательность I/O |
| **reg.c** | `register int` + цикл | Распределение по регистрам vs стек |
| **rot.c** | Ручной `rol()` на `unsigned` | Отдельная `sub_0010`, другая форма сдвигов |
| **st.c** | `struct Point` с `short` | Доступ к полям, смещение 0x21B в DGROUP/коде |
| **strcp.c** | `strcpy` из `<string.h>` | Вызов библиотечного `strcpy` vs инлайн; адрес литерала |

**Типичный паттерн:** семантика близка, но отличаются размер образа, адреса строк в DGROUP, порядок инструкций, выбор библиотечных vs инлайн-путей.

## 5. Рекомендации по приоритетам

1. **Декомпиляция:** не углубляться в тела несопоставленных library-процедур (`malloc` path); добавить `RCL`/`RCR`.
2. **Кодоген:** `MemExpr` → валидные lvalue (`*p`, глобалы); строки → литералы; `WhileOperation` → `while`; убрать утечку имён IR (`div op`).
3. **Far/near:** `MK_FP`, индексация far-массивов, видеопамять (`fptr`, `vol`).
4. **Makefile:** long/`%` → подключать нужные helper'ы из CRT (не только `SLIBCE.LIB`).
5. **EXE match:** даже при корректном C побайтовое совпадение маловероятно без фиксации адресов DGROUP; для CI достаточно семантического эквивалента или сравнения хеша кода.

## 6. Исключения из автотеста

Файлы из разделов 1–3 перечислены в `roundtrip_xfail.txt` и не участвуют в `QuickCProgramRoundTripTests` (ожидаемые сбои).

Диагностический прогон: `QuickCProgramProbeTests.ProbeAllPrograms_PrintSummary` (требует DOSBox-X + QuickC).
